Türchen 17: Dynamic Programming

In der Klausur kann maybe eine kurze Frage zu Dynamic Programming drankommen, deshalb eine Wiederholung. Es ist eine Methode, in der man Probleme in kleinere Teilprobleme zerlegt und die Lösungen dieser Teilprobleme dann speichert (Memoization). Wir machen das Basic Beispiel zu den Fibonacci-Zahlen.

Die Fibonacci-Formel lautet:

F(0) = 0
F(1) = 1
F(n) = F(n-1) + F(n-2) für n > 1

Unsere Aufgabe ist es, erstmal eine Funktion zu schreiben, die diese Formel rekursiv umsetzt. Also die Funktion ruft sich wieder selbst auf, das solltest du hinbekommen :)

fibonacci_naive = function(n) {
    # Code hier rein
}

Dynamic Programming - Seite 2

Lösung:

fibonacci_naive = function(n) {
    if (n == 0){
        return(0)
    }
    if (n == 1){
        return(1)
    }
    return(fibonacci_naive(n-1) + fibonacci_naive(n-2))
}

Das Problem bei dieser Funktion ist, das sie langsam ist, weil wir viele Werte mehrfach berechnen und wir es viel effizienter lösen können, indem wir Zwischenergebnisse abspeichern.

Deine nächste task ist es also eine Funktion zu schreiben, die Memoization nutzt.

fibonacci_memoized = function(n) {
    # Code hier rein
}

Als Tippp, benutze einen Vektor zum Speichern und pack an die erste und 2te position schonmal die werte rein. Danach musst du die Formel einfach nur mit den Vektor in einer for-Schleife lösen

Dynamic Programming - Seite 3

Lösung:

fibonacci_memoized = function(n) {
    memo = numeric(n+1)  # unsere Zwischenergebnisse
    memo[1] = 0
    if (n > 0){
        memo[2] = 1
    }
    
    for (i in 3:(n+1)) {
        memo[i] = memo[i-1] + memo[i-2]
    }
    return(memo[n+1])
}

Soooo, das berechnen ist jetzt viel effizienter. Beim Dynamic Programming used man neben der Memoziation auch noch oft den Bottom-Up Ansatz, indem kleinere Teilprobleme zuerst gelöst werden. Dieser Ansatz vermeidet dann Rekursion komplett.

Deine nächste Aufgabe wäre es jetzt also eine iterative Funktion zu schreiben die keine Rekurison benutzt und die Fibonacci-Zahlen berechnet.

fibonacci_bottom_up = function(n) {
    # Code hier rein
}

Hier brauchst du wieder ein Abfangen für die Fälle n=0 und n=1, dann braucht du eine previous und current und next variable, zusammen mit einer for schleife. Dann musst du es nur noch zusammenbasteln,

Dynamic Programming - Seite 4

Lösung:

fibonacci_bottom_up = function(n) {
    if (n == 0){
        return(0)
    }
    if (n == 1){
        return(1)
    }
    
    prev = 0
    curr = 1
    for (i in 2:n) {
        next = prev + curr
        prev = curr
        curr = next
    }
    return(curr)
}

Das wars für heuteee, als kleine Zusammenfassung zu Dynamic Programming haben wir das gelernt:

  • Teile und speichere: Zerlege Probleme in kleinere Teilprobleme.
  • Memoization: Speichere Lösungen zur Wiederverwendung.
  • Bottom-Up: Löse Probleme iterativ, um Effizienz zu maximieren.
